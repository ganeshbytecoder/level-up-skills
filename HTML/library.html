How to Use This Library

The Active Learning System

Reading is not mastery. Using it is mastery. Here is how to do both.

Week 1–2

First Read — All 5 Books

Read in order. Don't take notes. Read for the story and feeling. Let the images form in your mind.

  • Focus on narrative, not memorization
  • Highlight concepts that resonate
  • Read before sleep — stories consolidate overnight
Week 3

Active Re-read — Books 1 & 2

Book 1: Use the Lexicon table as a checklist. Book 2: Find 23 patterns in real code you know.

  • Rate each term 🟢/🟡/🔴
  • Write one sentence per 🟢 term
  • Build your Pattern Sightings document
Week 4

Architecture Audit — Book 3

Audit one real system you've built or worked on against every pattern in Book 3.

  • Does it follow 12-Factor? Which factor is violated?
  • Are Bounded Contexts explicit?
  • Do dependencies point inward?
  • Write down 3 architectural debts
Week 5

Failure Flash Cards — Book 4

Five recurring patterns from 20 outages. Make one card per pattern and use before every deploy.

  • Human error with no safeguard
  • Missing limits (rate, size, time)
  • Dependency failure with no fallback
  • Data inconsistency / race condition
  • Config change without staged rollout
Week 6

The Six Questions — Book 5

Print the Six Questions and use them before every event-driven feature you ship.

  • What if delivered twice?
  • What if arrives out of order?
  • What if consumer crashes halfway?
  • What if downstream is slow — not down?
  • What if event is missing?
  • What if I need to replay this topic?
Month 2–3

Daily Deliberate Practice

One pattern per day. Morning: read the entry. Day: find it in your work. Evening: one sentence.

  • 23 days: GoF patterns
  • 12 days: failure categories
  • 12 days: 12-Factor practices
  • 46 days: EDA failure patterns
